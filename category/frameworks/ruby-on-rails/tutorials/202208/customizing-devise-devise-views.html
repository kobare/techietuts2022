Customizing Devise Views

Techie     August 2022

Introduction

Devise works really great right out of the box but doesn’t look good. The pages are plain so you’d definitely want to replace them with your custom views. It’s pretty straight forward as there’s not that much of tinkering needed to get them working. Let’s see!


Prerequisites

  1. Ruby 3.2.0
  2. Rails 7.0.3


Part A: Creating a Rails 7 Project

Create a new rails project called devise_email by following this document: Creating A Rails 7 App: With esbuild, bootstrap and jquery.


Part B: Installing Devise

1 . Add devise gem in the Gemfile file.


2 . Bundle install


3 . Setup devise


4 . Generate user model


Part C: Creating Custom Views

1 . Generate devise views


2 . Rename devise folder to users in app/views.


3 . Turn scoped views on in config/initializers/devise.rb

Devise will automatically check whether the custom view exists and if so, serve that up.


4 . Include these external stylesheets in the head tag in views/layouts/application.html.erb

These contain styling and icons for the custom pages that we will be creating in the next step.


5 . Style up the Sign In page.

Open views/users/sessions/new.html.erb and replace the code with this code.

Important: We are using a raw form rather than the form helper. Its easier to style the form this way. A form helper generates the attributes: id=”new_user” action=”/users/sign_in” accept-charset=”UTF-8” method=”post”, input tag names & id, link urls and form authenticity token. Hence we need to add those manually for all the custom pages.
